Q1. Explain the Document Object Model (DOM) and describe how JavaScript interacts with it to dynamically update web page contents. (Max 5 Marks)

Student Answer:

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. It represents the document as a tree structure, where each node is an object representing a part of the document (like an element, text, or attribute).

JavaScript interacts with the DOM by using methods like `getElementById()`, `querySelector()`, and `createElement()` to access, modify, and add elements dynamically. This allows for real-time updates without reloading the page, which is fundamental to modern web development.

Long Answer Question (2/3)

Q2. Describe the purpose of React Hooks, specifically comparing the usage of `useState` and `useEffect`. (Max 8 Marks)

Student Answer:

React Hooks let you use state and other React features without writing a class. `useState` is used to add state variables to function components, returning the current state value and a function to update it. This is how components hold onto data that changes.

`useEffect` is a Hook that lets you perform side effects in function components, like data fetching, subscriptions, or manually changing the DOM. It runs after every render, but you can control when it runs by passing a dependency array.

Long Answer Question (3/3)

Q3. What is the concept of component lifecycle in a React class component? Give the names of two common lifecycle methods. (Max 2 Marks)

Student Answer:

The component lifecycle is the sequence of stages a component goes through from mounting (creation) to updating (re-rendering) and unmounting (destruction). It allows developers to run code at specific points in a component's existence.

Two common lifecycle methods are `componentDidMount` and `componentDidUpdate`.
